TAG <FORM> ..Esempio (Record di Codecharge)..

Questo tag permette di gestire l'input remoto di un utente verso il server. L'utente ha di fronte a se la scheda (FORM) e vi può inserire informazioni come testo o scelte obbligate. Una volta riempiti tutti i campi l'utente sottomette (SUBMIT) i dati al server con il metodo indicato e il server li elabora scatenando le reazioni connesse.

SINTASSI 

<FORM
  ACTION="serverURL"
  ENCTYPE="encodingType"  (Default : application/x-www-form-urlencoded)
  METHOD="GET"|"POST"
  NAME="formName"
  ONRESET="JScode"
  ONSUBMIT="JScode"
  TARGET="windowName"
>

..Campi di input ..

</FORM>

Attributi:

  • ACTION="url" specifica l'url del programma eseguito al momento della sottomissione della form al server.
  • METHOD="get|post" stabilisce il metodo usato per inviare al server le informazioni. Il valore di default è GET che appende i dati di seguito all'indirizzo dell'ACTION generando così problemi di troncamento (il limite di lunghezza dell’url è di 2048 bytes. Il metodo POST spedisce i dati attraverso una transazione http post (quindi come sequenza separata) e non ha limite teorico di grandezza dei dati inviati.
  •  TARGET="nome" stabilisce in quale finestra debba essere caricato il risultato del puntamento. Può assumere valori speciali :

TARGET="_blank" crea il link ad una nuova finestra vuota.

TARGET="_self" crea il link alla stessa finestra.

TARGET="_parent" crea il link alla finestra parente. Se non esiste il link viene creato verso la finestra stessa come per "_self".

TARGET="_top" crea il link alla stessa finestra ma visualizzandola a tutto schermo. 

  • ENCTYPE="tipo_codifica" serve a specificare il tipo di codifica utilizzato.
