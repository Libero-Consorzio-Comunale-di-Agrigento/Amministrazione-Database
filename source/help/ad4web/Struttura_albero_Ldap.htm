Struttura albero Ldap

Definizione

L'albero Ldap che può essere automaticamente gestito tramite Ad4 prevede la possibilità di avere un sottoramo dove sono presenti le persone (indicato nei registri sotto la voce RootUsers) e un differente sottoramo dove viene riportata la Struttura Organizzativa persone (indicato nei registri sotto la voce RootOU).

Sia nel caso di allineamento da LDAP verso SO4 (SO4Mapping) che in caso di allineamento da SO4-AD4 verso LDAP (LDAPMapping), le registrazioni su LDAP avranno la seguente architettura:

-(ou): Struttura Organizzativa Istuituzionale

|

|-(ou): XXX1[##YYY]##Des.Abb.Unità1

|  |

|  |-(group): XXX1[##YYY]##Des.Abb.Unità1

|  |  |

|  |  |- (user): Utente1

| |

|  |-(group): Desc.Ruolo1##XXX1[##YYY]

|  |  |

|  |  |- (user): Utente1

|  |

|  |-(group): Desc.Ruolo2##XXX1[##YYY]

|  |  |

|  |  |- (user): Utente2

|

|-(ou): XXX2[##YYY]##Des.Abb.Unità2

|

|-(group): XXX2[##YYY]##Des.Abb.Unità2

|  |

|  |- (user): Utente2

|

|-(group): Desc.Ruolo1##XXX2[##YYY]

|  |

|  |- (user): Utente2

|

|-(group): Descr.Ruolo3##XXX2[##YYY]

|  |

| |- (user): Utente1

 

In questo esempio viene mostrata una struttura in cui per ogni "ou" è registrato un "group" con lo

stesso nome e un numero aggiuntivo di altri gruppi composti dal nome del Ruolo utilizzato all'interno di SO4 e seguito dalla codifica della Unità Organizzativa.

La codifica delle Unità Organizzative viene realizzata con l'identificazione univoca prevista in SO4

che è composta da un codice (massimo 16crt), opzionalmente seguito dal codice della "AOO" (

anch'esso per un massimo di 16crt) . Il codice della "AOO" può essere omesso se non c'è ambiguità nella identificazione della Unità Organizzativa.

Il codice della Unità Organizzativa viene concatenato al resto del nome della "ou" con il simbolo delimitatore (indicato nei registri), come anche l'eventuale codice di "AOO" che può seguire il codice della Unità Organizzativa concatenato con il simbolo delimitatore.

Nell'esempio :

· "XXX1" e "XXX2" indicano il codice delle Unità Organizzative previste in SO4 per un

massimo di 16 caratteri.

· " [YYY] " indica il codice della "AOO" previste in SO4 per un massimo di 16 caratteri.

Con le parentesi quadre si esprime la possibilità che il codice possa essere omesso.

· "Des.Abb.Unità1" e "Des.Abb.Unità2" indicano la descrizione abbreviata delle Unità Organizzative previste in SO4 per un massimo di 20 caratteri.

· "Des.Ruolo1", "Des.Ruolo2" e "Des.Ruolo3" indicano il nome del Ruolo o Incarico previsto in SO4 per un massimo di 40 caratteri.

Gli "user" vengono assegnati ai "group" e non alle "ou" per evitare il problema di doppia

assegnazione ad "ou" (non ammessa in LDAP) nel caso in cui l'Ente volesse mantenere la sua

struttura architetturale specifica.

Il "group" con lo stesso nome della "ou" rappresenta la assegnazione alla Unità Organizzativa

"prevalente", mentre la assegnazione ad un "group" diverso indica la assegnazione "secondaria" alla stessa Unità Organizzativa.

In questo modo sarà possibile alla SO4Mapping andare a prelevare correttamente le informazioni per la assegnazione di Unità Organizzative "prevalenti" e "secondarie", e del relativo "Ruolo" o "Incarico" di assegnazione nella Unità Organizzativa.
