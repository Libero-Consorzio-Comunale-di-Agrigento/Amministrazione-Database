Autenticazione via LDAP

L'autenticazione degli utenti del Sistema Informativo Versione 4 viene, normalmente, effettuata grazie alla definizione dell'utente e relativa password in AD4 – Amministratore Database.

E' possibile, però, attivare l'autenticazione LDAP per uno specifico Progetto / Istanza / Modulo tramite le relative Caratteristiche di Accesso.

I parametri di configurazione del server LDAP devono essere definiti nel Registro sotto la chiave PRODUCT\LDAPCONFIG.

Parametri di configurazione LDAP

Server

Nome del server LDAP a cui connettersi per autenticare gli utenti.

Porta

Porta a cui connettersi.

Contesto

Il nodo dell'albero della directory LDAP in cui sono definiti gli utenti.

TipoAutenticazione

Tipo si autenticazione al server LDAP.

Il solo valore gestito ad ora è ClearText.

TimeOut

Tempo concesso per tentare la connessione al server LDAP.

Alla scadenza, se la connessione non è stata effettuata, viene tentata l'autenticazione via AD4.

 

Attualmente il parametro non è ancora gestito.

Mandatory

Vale yes se il login è obbligatoriamente da effettuare sul server LDAP e no se, in caso di malfunzionamenti sulla connessione LDAP, il login viene fatto sulla base dati AD4.

SyncType

Vale Update se la sincronizzazione dei dati tra il server LDAP avviene solo in aggiornamento (i.e. Un utente presente sul server LDAP ma non in AD4 non viene inserito in quest'ultima), InsertUpdate se invece la sincronizzazione vale anche per l'inserimento di nuovi soggetti.

DefaultInsertGroups

Elenco dei gruppi, separati da virgole, a cui l'utente eventualmente inserito in AD4 deve essere aggiunto.

I gruppi devono esistere nella tabella UTENTI (tipo 'G') altrimenti viene emessa una segnalazione di errore sulla tabella ACCESSI

SuffissoNominativo

 

Suffisso da accodare allo User Logon Name (su server Microsoft Active Directory) per effettuare il login son il solo User Logon Name e non con il common name.
