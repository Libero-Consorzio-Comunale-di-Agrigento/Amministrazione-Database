Parametri Configurazione LDAP

 

Definizione

Finestra per l'inserimento dei parametri di configurazione dell'eventuale server LDAP che si desidera utilizzare per l'autenticazione degli utenti.

L'autenticazione degli utenti del Sistema Informativo Versione 4 viene, normalmente, effettuata grazie alla definizione dell'utente e relativa password in AD4 – Amministratore Database.

E' possibile, però, attivare l'autenticazione LDAP per uno specifico Progetto / Istanza / Modulo tramite le relative Caratteristiche di Accesso.

E' possibile configurare i parametri di più server LDAP di cui uno principale (massimo 11); al momento dell'autenticazione di un utente per cui è prevista autenticazione tramite LDAP ad un applicativo del Sistema Informativo V4, viene tentata la connessione al server LDAP principale e, in caso di mancata risposta del server o se l'utente non esiste, viene tentata la connessione ai server alternativi in ordine di configurazione.

Di seguito vengono elencati i parametri di configurazione che devono essere specificati per ogni server LDAP (principale compreso) e quelli del solo server LDAP principale.

Parametri di configurazione LDAP

Server

Nome del server LDAP a cui connettersi per autenticare gli utenti.

Porta

Porta a cui connettersi.

Contesto

Il nodo dell'albero della directory LDAP in cui sono definiti gli utenti.

UtenteConnessioneLDAP

Utente che può accedere all'albero LDAP definito nella stringa Contesto.

PwdConnessioneLDAP

Password di UtenteConnessioneLDAP.

SuffissoNominativo

 

Suffisso da aggiungere al nominativo dell'utente (per Active Directory è necessario perchè ci si autentica tramite userPrincipalName invece che tramite CommonName).

ContestoGruppi

Contesto dell'albero LDAP in cui cercare i gruppi da mappare.

Parametri di configurazione aggiuntivi LDAP Principale

SyncType

Tipo di sincronizzazione prevista per l'allineamento degli utenti:

·      Update: autentica solo gli utenti già presenti in AD4 e aggiorna la password;

·      InsertUpdate: inserisce gli utenti non presenti in AD4 e aggiorna la password di quelli esistenti.

Mandatory

Indica se l'autenticazione via LDAP è obbligatoria.

·      Yes: in caso il server LDAP non risponda, l'utente non si autentica;

·      No: in caso il server LDAP non risponda, viene tentata l'autenticazione dell'utente tramite AD4.

TipoAutenticazione

Tipo di autenticazione da effettuare sul server LDAP.

·      ClearText

·      SSL

DefaultInsertGroups

Gruppi di AD4 separati da virgola in cui inserire gli utenti presenti in LDAP e non in AD4.

I gruppi devono esistere (tabella UTENTI – tipo 'G') altrimenti viene emessa una segnalazione di errore sulla tabella ACCESSI.

AuthorizationMapping

Indica se deve essere effettuato l'allineamento dei gruppi LDAP mappati in AD4 (Valori possibili: yes/no - default: no).

SO4Mapping

Indica se deve essere effettuato l'allineamento della struttura da LDAP verso AD4 e SO4 (Valori possibili: yes/no - default: no)

LdapMapping

Indica se deve essere effettuato l'allineamento della struttura da AD4 verso LDAP (Valori possibili: yes/no - default: no)

TimeOut

Tempo concesso per tentare la connessione al server LDAP.

Alla scadenza, se la connessione non è stata effettuata, viene tentata l'autenticazione via AD4.

Attualmente il parametro non è gestito.

I vari tipi di allineamento possibili sono fra loro esclusivi quindi solo uno può essere attivo fra AuthorizationMapping, SO4Mapping e LdapMapping.

Parametri specifici per LdapMapping

Configurabili da Servizi à Configurazione LDAP

RootOU

Radice nella quale inserire la struttura delle unità organizzative.

RootUsers

Radice dalla quale ricercare un utente eventualmente già presente in struttura.

RootUsersDisabled

Radice nella quale inserire un utente non più presente in struttura. Se non valorizzato, usa RootUsers.

Delimitatore

Usato in scrittura dell'albero LDAP. NON AMMESSI: "," "=" "+" "<" ">" "#" ";" "/"

Parametri relativi ad attributi da mappare in LDAP in LdapMapping.

Configurabili da Servizi à Registro

Indicare nel sottoramo PRODUCTS/LDAPCONFIG/ATTRIBUTI tutti gli attributi che si vogliono mappare nell'albero Ldap.

Indicare nel sottoramo PRODUCTS/LDAPCONFIG/ATTRIBUTI\CREAZIONE solo gli attributi che devono essere valorizzati solo in fase di creazione di un utente Ldap.

Il nome del campo deve essere un campo esistente in Ldap.

Variabili Utilizzabili:

%utente%: utente in AD4

%username%: nominativo di login dell'utente

%nome% : nome del soggetto associato all'utente

%cognome% : cognome del soggetto associato all'utente

%email% : indirizzo web del soggetto associato all'utente

È possibile indicare attributi che devono essere calcolati mettendo come primo carattere "=".

Ad esempio:

stringa: HomePhone

valore: =soggetto.get_telefono(utente.get_soggetto('%UTENTE%','N',0))

Per valorizzare il campo ldap HomePhone verrà sostituito a %UTENTE% l'utente che si è connesso e verrà eseguita la funzione per determinare il valore da assegnare.

Funzionalità particolari Parametri

Crea alternativo

Funzione attiva unicamente sul server principale.

Consente di inserire i parametri di configurazione di un server alternativo.

Elimina alternativo

Funzione disponibile sui soli server alternativi.

Consente di eliminare tutti i parametri di configurazione del server alternativo selezionato.

 

E' possibile, inoltre, definire un'associazione tra gruppi LDAP e gruppi AD4 nella pagina "Configurazione LDAP > Collegamento Gruppi".

 

Una volta definiti i parametri di configurazione, perché l'autenticazione avvenga tramite LDAP è necessario specificarlo nelle caratteristiche di accesso dei

- Progetti  oppure

- Istanze  oppure

- Moduli  oppure

- Accessi di Gruppi di Utenti oppure

- Accessi dei singoli Utenti

per cui si intende attivarla.
