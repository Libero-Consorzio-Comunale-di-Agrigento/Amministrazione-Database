Eventi

Definizione

Traccia degli eventi e delle eventuali segnalazioni registrati dagli applicativi del Sistema Informativo V4.

La visualizzazione degli eventi può essere attivata dal menu "Servizi > Eventi".

Gli eventi possono essere filtrati tramite il tipo e vengono elencati in ordine di data (e ora) di registrazione decrescente. L'utente può comunque (tramite la funzione ricerca) impostare dei criteri di ricerca più restrittivi.

Il filtro iniziale sul tipo non elenca tutte le tipologie di evento ma soltanto dei macro sottoinsiemi:

1.    Errore

2.    Archiviazione Accessi

3.    Messaggi applicativi

4.    Tutti

Gli eventi che rappresentano le registrazioni di accesso sono visibili dal menu "Servizi > Accessi" (oltre che, ovviamente, selezionando tipo di evento: "Tutti").

Gli eventi di tipo "Archiviazione Accessi" rappresentano eventi scatenati dall'archiviazione della traccia degli accessi (attivabile dalle caratteristiche di accesso di ogni singolo progetto od istanza); se l'evento ha un file associato presente sul file system del server è presente l'immagine che permette di aprirlo.

E', inoltre, possibile selezionare soltanto gli eventi cui sia associato un file di archiviazione tramite il filtro "Archiviazione - solo file".

Dati

Evento

Numero di evento.

Data

Data e ora di registrazione dell'evento.

Tipo

Tipo di evento.

·      Accesso Attivo (LOGON) / Chiuso (LOGOFF): connessione al modulo per una sessione Attiva / Chiusa;

·      Accesso Funzione (LOGTRC): registrazione del lancio di funzionalità attivate da menu;

·      Accesso Uscito (LOGOUT): chiusura della sessione di lavoro;

·      Accesso Uscito con Errore (LOGABT): uscita dalla procedura con errore bloccante (System Error);

·      Accesso Password Errata (LOGWPW): tentativo di accesso al modulo da parte di un utente che ha immesso una password sbagliata;

·      Errore (ERROR): eventi di errore registrati dagli applicativi;

·      Archiviazione Accessi (ARCLOG): eventi scatenati dall'archiviazione della traccia degli accessi (archiviazione avvenuta, ripristino accessi archiviati);

·      Messaggi applicativi (APPTRC): segnalazioni derivanti da elaborazioni di altri applicativi (ad es. risultati di elaborazioni batch).

Testo / Note

Descrizione dell'evento.

Informazioni aggiuntive.

Gravità

Indica la gravità dell'evento.

·       Informazione;

·       Esclamazione (avvertimento);

·       Stop (segnalazione bloccante).

Provenienza

Nominativo UTENTE del sistema SI4.

Codice – Descrizione ISTANZA di connessione.

Codice – Descrizione MODULO di connessione.

DB User

Utente di database su cui era effettuata la connessione al verificarsi dell'evento.

Stato

Stato della registrazione.

·       In uso: registrazione in linea sul database e mai archiviata su file.

·       Da archiviare: l'elaborazione di archiviazione sta trattando la registrazione per trasferirla su file.

·       Archiviato: la registrazione è già stata archiviata su file e successivamente ripristinata (rimessa in linea sul database).

Notificato

Indicatore dell'avvenuta notifica dell'evento.

Funzionalità particolari

Elimina

Funzione attiva in multiselezione.

Gli eventi che rappresentano le registrazioni di accesso (per cui è prevista la memorizzazione dei dati su file o la cancellazione dopo un numero di giorni definito dall'ente) NON sono eliminabili.

Ripristina Accessi

Funzione disponibile sugli eventi di tipo "Archiviazione Accessi" che notificano l'avvenuta archiviazione degli accessi su file (sfondo verde).

Consente di ripristinare sul database le registrazioni già archiviate su file. Dopo il ripristino, le registrazioni sono visualizzabili tramite menu "Servizi -> Accessi".

Visualizza Log

Funzione disponibile sugli eventi di tipo "Archiviazione Accessi" che notificano l'avvenuta archiviazione degli accessi su file (sfondo verde).

Consente di visualizzare il file xml in cui sono state archiviate le registrazioni.
